Индивидуальный вклад

Ответственность за backend web-игры: на основе документации по PostgreSQL (хранимые функции и процедуры user_*, task_*, suggestion_*, achievements_*) и требований к проекту спроектирована архитектура Controller → DAO → PostgreSQL. Реализован REST API на Kotlin + Spring Boot 3: модули пользователей, задач, предложений и достижений. Для каждого домена — контроллеры с HTTP-ручками, DAO с JDBC-доступом, модели данных, константы SQL-запросов (DBQueries*) и имён полей БД (FieldConstants). Настроено подключение к PostgreSQL через DBConnection и application.properties. Добавлена валидация входных данных (например, полей регистрации и авторизации) и обработка доменных ошибок с корректными HTTP-кодами — в том числе для сценария решения задачи (неверный ответ, задача уже решена, пользователь/задача не найдены).

Написаны модульные и интеграционные тесты (JUnit 5, MockMvc, Mockito): сценарии для UserController и валидатора полей пользователя. Подготовлена пользовательская документация (ReadMe.md) — запуск приложения, настройка БД, описание всех API-ручек с примерами запросов.

Сложности и их решение: при работе с DAO обнаружена ошибка — общее JDBC-соединение закрывалось после первого запроса из‑за connection.use { }, из‑за чего последующие обращения к API падали с PSQLException: Соединение уже было закрыто. Исправлено: .use применяется только к PreparedStatement и ResultSet, соединение остаётся открытым на время жизни приложения. Другая сложность — исключения PostgreSQL из процедуры task_solve (например, «Неверный ответ для задачи») пробрасывались клиенту как необработанные 500-ошибки. Добавлен маппинг сообщений БД в доменные исключения (WrongTaskAnswerException, TaskAlreadySolvedException и др.) и обработка в TaskController с возвратом 400, 404, 409. При интеграции с БД также учтено расхождение имён колонок (id vs task_id) — приведены в соответствие константы полей и маппинг в DAO.

Результат: работающий backend-сервис с REST API для клиента (регистрация/авторизация, каталог и решение задач, предложения игроков, достижения), покрытый тестами и документацией, готовый к интеграции с frontend.
